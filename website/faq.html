FAQ

Questions, answered.

The honest version — the same answers you'd find reading the project's README.

Is Omega Client free?

Yes. The launcher and companion mod are free and open source under the MIT license. Nothing needed to launch, mod, or play is ever paywalled. Development is funded by optional, clearly-disclosed sponsor placements and optional vanity cosmetics — neither gates any functionality.

Do I need to own Minecraft?

Yes. Omega requires signing in with a Microsoft account that owns Minecraft: Java Edition — the same requirement as Lunar, Feather, and every legitimate launcher. There is no offline or cracked mode. Sign-in uses Microsoft's standard public OAuth flow, works out of the box, and your tokens are stored encrypted on your machine using your OS keychain.

Is it allowed on servers? Is this a hack client?

Omega is not a hack client. Every built-in feature is a visual or convenience setting: fullbright, FOV/zoom, toggle sprint, HUD info, particle control, and similar. There is deliberately no reach, no aim assist, no auto-clicker, no velocity modification, and no X-ray — the block highlighter is depth-tested and will never show blocks through walls.

That said, a few servers restrict even visual settings like fullbright. Rules differ, so check the rules of wherever you play.

Windows says "Windows protected your PC" / macOS won't open the app. Is it safe?

The builds are not code-signed (signing certificates cost money the project doesn't spend), so operating systems show their standard unknown-publisher warnings. On Windows, click More info → Run anyway. On macOS, right-click the app → Open once — it opens normally after that. On Linux, chmod +x the AppImage first.

If you'd rather not trust a prebuilt binary, the entire project is open source — you can read the code and build it yourself. Every release file also lists its sha256 digest on the release page.

Does Omega include mods? Where do mods come from?

You bring your own mods — Omega imports your .jars and manages them as toggles. The only things installed for you are the Omega companion mod itself and, when an instance needs them, three well-known dependencies fetched from Modrinth: Fabric API, and a shader loader (Iris + Sodium on Fabric, Oculus on Forge). Nothing else is bundled or downloaded, and nothing is re-hosted.

Will it mess with my existing .minecraft folder?

No. Omega works on top of any standard install — official launcher, MultiMC, Prism, or manual. Disabling a mod renames it to mod.jar.disabled (which every loader ignores) rather than moving or deleting anything. Installs download only missing files, sha1-verified, straight from Mojang's own CDNs.

Which Minecraft versions and loaders are supported?

The launcher can install and launch any Minecraft release, vanilla or with Fabric/Forge on top, and auto-detects existing installs of vanilla, Forge, Fabric, Quilt, and NeoForge. The built-in Omega companion mod currently targets Minecraft 1.20.1 (Fabric and Forge builds) — on other versions you still get the full launcher, just without the in-game Omega menu.

How do updates work?

Builds installed via the Windows installer, the macOS app, or the Linux AppImage check the rolling release on startup, download updates in the background, and apply them on the next restart — a banner offers "Restart now". The check is configurable in Settings → Updates. The Windows portable .exe can't replace itself; portable users just re-download.

Can I use more than one account?

Yes. Link as many Microsoft accounts as you like and pick which one an instance uses via the account switcher next to Play. Minecraft can't hot-swap accounts mid-game, so the in-game Switch Account button cleanly quits and pops the launcher back up with the switcher open — pick the next account and relaunch.

How do shaders work?

A compatible shader loader is preinstalled automatically — Iris + Sodium on Fabric, Oculus on Forge. Import .zip shader packs in the launcher's Shaders tab, then pick the active pack in-game under Video Settings → Shader Packs (that menu lives inside the game, so the launcher can't select it for you).

What data does Omega collect?

None — there is no launcher backend of any kind. Installs come straight from Mojang's and the loaders' own servers (plus Modrinth for the three bundled dependencies), and authentication talks directly to Microsoft/Xbox/Mojang, nowhere else. There's no telemetry, no analytics, and no account system beyond your own Microsoft sign-in, which stays on your machine.

Something broke — where do I get help?

Open an issue on GitHub with what you did, what happened, and the output from the launcher's Console tab if the game failed to start. If a launch fails with missing files, the fix is usually one click: re-run Install on the instance to repair it.

Still curious? Just try it.

It's free, it's small, and it won't touch your existing setup.
